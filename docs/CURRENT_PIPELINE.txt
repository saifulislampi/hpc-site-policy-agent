  → calculate deterministic topic coverage

       → validate every tool call and selected URL locally
       → stop at finish_discovery or max_steps
  → construct canonical web corpus records
       → preserve URL, scope, trust, headings, hashes, and fetched timestamp
       → keep tables atomic as Markdown
       → keep Policies and FAQ sections whole
       → split ordinary text at natural boundaries
       → overlap_chars = 0
       → skip PDFs for corpus retrieval
  → merge into corpora/<site-id>/
       → manifest.json
       → documents.jsonl
       → chunks.jsonl
       → never persist a retrieval index

       → pages not rediscovered remain stored
  → build a transient CPU BM25 index
  → retrieve separately for every extraction field
       → use several lexical query variants per field
       → site_scope is an input filter before ranking
       → allow target_site + applicable organization_general
       → exclude sibling + unrelated
       → deduplicate byte-identical content from alternate URLs
       → suppress navigation noise and field-incompatible hardware tables
       → include charging_model + purge_policy + cost_traps

       → retry one complete extraction on validation failure using the same model
