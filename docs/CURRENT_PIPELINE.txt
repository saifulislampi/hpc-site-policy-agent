HPC SITE POLICY AGENT — CURRENT PIPELINE

CLI arguments + environment
  → derive provider-neutral site identity
  → generate bounded canonical-root + policy searches
       → submission and partitions
       → networking
       → FAQ, charging/accounting, purge, retention, and storage policy
  → search approved organization domains
  → classify scope deterministically from URL path / manifest-equivalent identity
       → target_site
       → organization_general
       → sibling
       → unrelated
     and assign trust independently
       → official_web
       → manifest_declared (reserved)
       → unverified_local (reserved)
  → rank URL/title topic evidence above noisy search snippets
  → fetch canonical root + several high-value pages per topic
       → reject already-attempted URLs
       → continue after redirects, failures, or topic mismatches
       → prefer article content over navigation chrome
  → retain a fetched sibling page as a negative control when budget permits
  → calculate deterministic topic coverage
  → bounded provider discovery loop
       → search_web | fetch_page | finish_discovery
       → validate every tool call and selected URL locally
       → stop at finish_discovery or max_steps
       → if the model loop fails/times out, retain deterministic crawl state
         and continue with a partial-discovery selection
  → construct canonical web corpus records
       → preserve URL, scope, trust, headings, hashes, and fetched timestamp
       → keep tables atomic as Markdown
       → keep Policies and FAQ sections whole
       → split ordinary text at natural boundaries
       → overlap_chars = 0
       → skip PDFs for corpus retrieval
  → merge into corpora/<site-id>/
       → manifest.json
       → documents.jsonl
       → chunks.jsonl
       → never persist a retrieval index
       → --refresh-corpus replaces changed rediscovered pages
       → pages not rediscovered remain stored
  → build a transient CPU BM25 index
  → choose extraction profile
       → site-policy (default): operational submission + network fields
       → evaluation-full: default fields + secondary operational policies
  → retrieve separately for every profile-requested field
       → use several lexical query variants per field
       → site_scope is an input filter before ranking
       → allow target_site + applicable organization_general
       → exclude sibling + unrelated
       → deduplicate byte-identical content from alternate URLs
       → suppress navigation noise and field-incompatible hardware tables
       → evaluation-full includes charging_model + purge_policy + cost_traps
       → retain whole Policies/FAQ candidates for those fields
  → split retrieved text into exact, zero-overlap evidence spans
       → expose field-local references such as partitions:E4
       → model selects span IDs instead of copying quotes or provenance
       → Python inserts the literal span, canonical chunk ID, URL, and heading
  → independent provider structured-extraction calls
       → submission group
       → network group
       → operational group only for evaluation-full
       → all groups use the single CLI --model
  → validate each field independently
       → chunk_id belongs to that field's retrieved list
       → evidence URL matches the chunk URL
       → quote is a literal substring of the chunk
       → preserve valid fields
       → retry only fields with invalid evidence references once
       → on timeout/invalid retry, set affected values to null
         with documentation_status=extraction_failed
       → fields outside the profile are null/not_investigated
  → log every retrieved chunk, score, citation state, and retrieved-but-uncited ID
  → build detailed discovery report
       → run, discovery, source scope/trust, corpus fingerprint
       → field retrieval audit + scores
       → findings + chunk/source evidence
       → sibling retained/retrieved counts
       → profile, requested/not-investigated/unverified/failed fields
       → group errors and retried fields for evaluation
  → build compact candidate site policy
       → consumer-ready values only
       → profile_state=complete|partial
       → provenance references detailed findings + corpus fingerprint
  → write timestamped report + site-policy JSON
  → write JSONL execution trace
